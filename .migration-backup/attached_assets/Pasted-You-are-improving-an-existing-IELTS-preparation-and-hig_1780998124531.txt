You are improving an existing IELTS preparation and higher study tracker web app called "Within a Few Weeks". The app is built with React, TypeScript, Vite, Tailwind CSS, shadcn/ui, TanStack Query, and Wouter for routing. The backend is Vercel serverless functions (api/index.mjs) connected to a Neon PostgreSQL database via drizzle-orm/neon-http.

IMPORTANT: Do NOT touch or modify these files:
- api/index.mjs
- vercel.json
- .vercelignore
- lib/db/src/index.ts
- Any file in lib/db/src/schema/
- pnpm-workspace.yaml
- Any backend/server files

Only modify files inside: artifacts/ielts-tracker/src/

The existing database tables already available are:
- settings (id, name, examDate, targetReading, targetListening, targetWriting, targetSpeaking, dailyGoalMinutes, darkMode)
- scores (id, date, module, score, band, notes, createdAt)
- study_sessions (id, date, module, minutes, activityType, wentWell, improve, createdAt)
- practice_logs (id, date, module, subType, score, totalQuestions, notes, createdAt)
- vocab_words (id, word, pos, definition, example, topic, known, createdAt)
- favourite_affirmations (id, affirmation, savedAt, createdAt)
- higher_study_applications (id, universityName, country, program, degreeType, status, deadline, appliedDate, notes, reqSop, reqLor1, reqLor2, reqLor3, reqTranscripts, reqCv, reqGre, reqToefl, reqPortfolio, requirementsJson, createdAt)
- other_test_scores (id, testName, attemptDate, totalScore, sectionsJson, notes, createdAt)
- scholarships (id, name, provider, amount, currency, fundingType, deadline, status, notes, createdAt)
- checklist_templates (id, name, degreeType, items, createdAt)

All API calls go through artifacts/ielts-tracker/src/lib/api.ts using fetch('/api/...').
