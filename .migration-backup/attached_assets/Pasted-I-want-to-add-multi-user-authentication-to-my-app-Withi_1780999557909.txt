using any third-party auth service like Clerk or Auth0. Build everything from 
scratch using JWT tokens and bcrypt, with users stored in Neon PostgreSQL.

The stack is:

- Backend: Vercel serverless function (api/index.mjs) - single file handler
- Database: Neon PostgreSQL with drizzle-orm/neon-http and @neondatabase/serverless
- All DB schema is in lib/db/src/schema/ as individual .ts files
- All API routes are handled in api/index.mjs as URL pattern matching

IMPORTANT - Do NOT touch these files:

=== STEP 1 - Add users table ===

Create a new file lib/db/src/schema/users.ts:

import { pgTable, serial, text, timestamp } from "drizzle-orm/pg-core";

export const usersTable = pgTable("users", {
  id: serial("id").primaryKey(),
  name: text("name").notNull(),
  email: text("email").notNull().unique(),
  passwordHash: text("password_hash").notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
});

export type User = typeof usersTable.$inferSelect;

Then add this export to lib/db/src/schema/index.ts:
export * from "./users";

Then run: pnpm --filter @workspace/db run push
to create the users table in Neon.

const usersTable = pgTable("users", {

Add a JWT helper function:
function verifyToken(req) {
  const auth = req.headers.authorization;
  if (!auth || !auth.startsWith("Bearer ")) return null;
  try {
    const token = auth.slice(7);
    return jwt.verify(token, process.env.JWT_SECRET);
  } catch {
    return null;
  }
}

Add these new API routes BEFORE the existing routes in the handler:

POST /api/auth/register:
- Validate: name (string), email (string), password (min 6 chars) using zod
- Check if email already exists in usersTable
- If exists: return 409 { error: "Email already registered" }
- Hash password: const hash = await bcrypt.hash(password, 10)
- Insert new user into usersTable
- Generate JWT: jwt.sign({ userId: user.id, email: user.email }, process.env.JWT_SECRET, { expiresIn: "7d" })
- Return: { token, user: { id, name, email } }

POST /api/auth/login:
- Validate: email, password using zod
- Find user by email in usersTable
- If not found: return 401 { error: "Invalid email or password" }
- Compare password: await bcrypt.compare(password, user.passwordHash)
- If wrong: return 401 { error: "Invalid email or password" }
- Generate JWT same as register
- Return: { token, user: { id, name, email } }

GET /api/auth/me:
- Call verifyToken(req)
- If null: return 401 { error: "Unauthorized" }
- Find user by id from token
- Return: { id, name, email }

For ALL other existing routes (settings, scores, study-sessions, etc.):
- Call verifyToken(req) at the start
- If null: return 401 { error: "Unauthorized" }
- Extract userId from the decoded token
- For GET: add .where(eq(table.userId, userId)) to filter by user
- For POST: add userId to the inserted values object
- For PUT/PATCH: verify row belongs to userId
- For DELETE: verify row belongs to userId before deleting

For settings specifically:
- Use userId.toString() as the settings id instead of "default"
- So GET /api/settings looks for eq(settingsTable.id, userId.toString())
